# Skill: fix-development

## 触发条件

当收到 bug 报告或发现代码缺陷时调用本 skill。

## 输入

```
bug_description: 缺陷描述（预期行为 vs 实际行为）
context: 涉及的文件/模块（可选）
```

## 执行步骤

```
Step 1. 复现问题
  - 找到触发缺陷的最小条件
  - 确认预期行为 vs 实际行为

Step 2. 定位根因
  - 读取涉及的代码文件
  - 分析错误类型：逻辑错误/边界条件/依赖问题/API 不兼容

Step 3. 制定修复方案
  - 确认最小修复范围（只改必要代码）
  - 评估回归风险

Step 4. 实现修复
  - 修改代码
  - 验证修复有效

Step 5. 输出 Fix 报告（见输出格式）

Step 6. 写入 910-skill-run-log.md
```

## 输出格式

```markdown
## Fix 报告

**缺陷**：{一句话描述}
**根因**：{根本原因}
**修复**：{修复方案}
**涉及文件**：{文件列表}
**验证**：{如何确认修复有效}
```

## 成功标准

- 缺陷已修复并验证
- 写入了 910-skill-run-log.md
